# 執筆支援テンプレート

Claude Code を活用した執筆支援のためのテンプレートリポジトリです。

## 概要

アイディアの発散から原稿の完成まで、執筆ワークフロー全体をAIが支援します。

## セットアップ

1. このテンプレートから新しいリポジトリを作成
2. [Claude Code](https://claude.ai/code) をインストール
3. プロジェクトディレクトリで `claude` を起動

## ディレクトリ構成

```
ideas/          # アイディアメモ
research/       # 調査・リサーチ
drafts/         # 原稿
project.md      # プロジェクト情報（タイトル、概要）
outline.md      # アウトライン
```

## コマンド

| コマンド | 説明 |
|----------|------|
| `/idea` | アイディアを記録するファイルを作成 |
| `/reflect` | アイディアをアウトラインや設定に反映 |
| `/research` | トピックをバックグラウンドで調査 |
| `/draft` | アウトラインをもとに原稿の草案を作成 |
| `/feedback <file>` | 草案にフィードバック用のアイディアファイルを作成 |
| `/proofread` | 誤字脱字・文法をチェック（校正） |
| `/review` | 内容の整合性・事実確認（校閲） |
| `/progress` | 文字数・ページ数・進捗を表示 |

## ワークフロー

```
/research ⇄ /idea → /reflect → /draft → /proofread → /review
              ↑         │         │
              └─────────┴─────────┘
                 /feedback (outline.md / drafts/*.md → /idea へ戻す)
```

起点は `/research` でも `/idea` でも構いません。リサーチから得たインサイトを `/idea` で書き留めて往復しながら材料を集めます。

1. **調査・リサーチ**: `/research` で必要な情報をバックグラウンドで収集
2. **アイディアの発散**: `/idea` で思いついたことやリサーチから得たインサイトを記録
3. **アイディアの反映**: `/reflect` でアウトラインや各種ファイルへ整理
4. **原稿の執筆**: `/draft` で草案を作成（または手動で執筆）
5. **校正**: `/proofread` で誤字脱字・表記をチェック
6. **校閲**: `/review` で内容の整合性を確認

`/draft` は省略でき、アウトラインが固まったら自力で書き上げる運用も可能です。

### Tips: /feedback でアウトラインや草案をブラッシュアップ

`/feedback` は対象ファイルの各行を引用形式にしたアイディアファイルを作成します。引用にコメントを書き加えて `/reflect` を実行すると、コメントが該当ファイル（や関連ファイル）に反映されます。

このループは**アウトライン段階・草案段階のどちらでも**使えます。

アウトライン段階：

```
/reflect → /feedback outline.md → (コメント追記) → /reflect → ...
```

草案段階：

```
/draft → /feedback drafts/01.md → (コメント追記) → /reflect → /draft → ...
```

`/feedback` の出力例：

```markdown
> 吾輩は人間である。
> 名前もある。
```

フィードバックしたい行だけを残してコメントを追記し、`/reflect` で反映させます：

```markdown
> 吾輩は人間である。
「ヒトである」の方が適切かも。アウトラインに追記しましょう。
```

このフィードバックループを繰り返すことで、アウトプットが段階的に洗練されていきます。

## フィクション執筆の場合

人物設定や舞台設定を管理したい場合は、`settings/` ディレクトリを作成してください。

```bash
mkdir -p settings/characters settings/world
```
