# 校閲

原稿の内容について、整合性・事実確認・論理構成をチェックします。

## 対象の指定

$ARGUMENTS

- 引数がある場合：指定されたファイルを校閲
- 引数がない場合：`drafts/` 内のすべてのファイルを対象とする

## チェック項目

### 内部整合性
- 登場人物の名前・設定の一貫性
- 時系列の矛盾
- 伏線の回収漏れ
- 場所・距離・時間の整合性

### 事実確認（ノンフィクション・専門的内容）
- 固有名詞の正確性
- 歴史的事実との整合
- 専門用語の適切な使用
- 引用・参照の正確性

### 論理構成
- 論旨の一貫性
- 段落間のつながり
- 結論と根拠の対応

### 設定との整合（フィクション）
- `settings/` に定義された設定との矛盾
- キャラクターの言動の一貫性
- 世界観のルールとの整合

## 出力形式

```
## 校閲結果：[ファイル名]

### 整合性の問題

1. **行XX**: ○○について
   - 問題：△△と矛盾しています
   - 参照：[関連ファイル]の記述
   - 提案：□□に修正する

### 事実確認

1. **行XX**: 「○○」について
   - 確認結果：正確 / 要確認 / 誤り
   - 備考：△△

### 構成への提案

- ○○の段落は、△△の後に移動した方が流れが良くなります

---

指摘事項について確認しますか？
```

## 注意

- 事実確認で不明な点は「要確認」として明示し、断定しないこと
- 設定との整合性チェックには `settings/` や `outline.md` を参照すること
- 提案は根拠とともに示すこと
- 作者の意図的な表現の可能性がある場合は、確認を取ること
