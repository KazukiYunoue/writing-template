# アイディアの反映

`ideas/` 配下のアイディアファイルを読み取り、適切なファイルへの反映を計画してください。

## 対象ファイルの指定

$ARGUMENTS

- 引数がある場合：指定されたファイルのみを対象とする
- 引数がない場合：`status: new` のアイディアファイルをすべて対象とする

## 手順

1. 対象のアイディアファイルを読み込む
2. 既存のファイル構成を確認する（outline.md, research/, settings/ など）
3. アイディアの内容を分析し、以下のいずれかに分類する：
   - タイトル・概要・ログライン → `project.md`
   - アウトラインへの追記・修正 → `outline.md`
   - 調査が必要な事項 → `research/` に新規ファイルまたは追記
   - 人物・舞台設定（フィクションの場合） → `settings/` 配下
   - その他の構造化された情報 → 適切なファイルを提案
   - `@claude` で始まる行 → Claude への指示（プロンプト）として解釈し、内容に応じた処理を反映計画に含める
4. 反映計画をユーザーに提示し、承認を待つ
5. 承認後、反映を実行し、アイディアファイルの `status` を `reflected` に更新

## 出力形式

```
## 反映計画

### [アイディアファイル名]

**要約**: （アイディアの要点）

**反映先**:
1. `outline.md` - ○○の章に△△を追加
2. `research/xxx.md`（新規作成） - □□について調査メモ

---

この計画で反映してよろしいですか？
```

## `@claude` 行の処理ヒント

- 時間のかかる作業（リサーチなど）はバックグラウンドのサブエージェントへ委譲し、完了は待たない
- 質問・相談などは `/reflect` の計画提示の中で回答する。アイディアファイルには書き戻さない
- `@claude` 行は処理後も削除せず履歴として残す

## 注意

- 必ずユーザーの承認を得てから反映すること
- 既存の内容を勝手に削除・大幅変更しないこと
- 反映先が不明確な場合は、ユーザーに確認すること
